# 极简项目管理（PMP）Skill

## 快速开始（默认做法）

1. 先用“如来十掌”把需求问全、把边界说清（见 `references/methodology.md`）。
2. 立刻产出 **一页项目任务书**（章程/任务书）作为对齐与授权的“契约”（模板见 `references/templates.md`）。
3. 用 **相关方登记册 + 权力-利益方格** 做期望与阻力管理（模板见 `references/templates.md`）。
4. 用 **三落实 + RAM/RACI** 把责任压实到人（模板见 `references/templates.md`）。
5. 进入规划与执行：WBS → 里程碑/关键路径 → 预算/投入 → 风险 → 状态报告 → 变更受控 → 收尾复盘。

如果用户希望“落到文件上”，优先运行脚本一键生成项目文档骨架：

```bash
python3 skills/pmp-jijian-project-management/scripts/init_project_pack.py --dir ./my-project --name "项目名"
```

## 核心框架（来自《极简项目管理》）

- **五大过程组**：启动 / 规划 / 执行 / 监控 / 收尾
- **如来十掌（10个知识领域/核心问题）**：
  1) 范围（做什么） 2) 进度（何时做完） 3) 成本（花多大代价）
  4) 质量（做到什么程度） 5) 资源（谁/什么资源做）
  6) 采购（资源不足如何外包/采购） 7) 沟通（怎么协同）
  8) 相关方（谁影响成败、如何管理期望） 9) 风险（不确定性）
  10) 整合（在相互竞争目标下做最优权衡）

## 工作流（按交付物驱动）

按下面顺序推进；每一步都要明确：**产出物、责任人、验收标准、节奏**。

1. **启动**
   - 产出：一页任务书、相关方登记册、权力-利益方格、RAM/RACI、启动会要点
2. **规划**
   - 产出：WBS（面向可交付成果）+ WBS词典、里程碑/关键路径（必要时）、预算与投入核对、风险登记册、沟通节奏、质量基准
3. **执行**
   - 产出：任务墙/看板、周节奏（例会+状态报告）、问题清单与决策记录、跨部门协调方案
4. **监控**
   - 产出：状态报告（偏差分析+预测）、变更申请与评审记录（九步法）、纠偏行动项
5. **收尾**
   - 产出：验收记录、收尾报告（至少五部分）、经验教训（可落实）、归档清单与复用机制

## 该读哪些参考文件（按需加载）

## 输出要求（对另一个 Codex 的约束）

- 默认输出为 **Markdown**，并尽量“可复制粘贴就能用”。
- 任何计划/表格都要能落到：**谁（Owner）在什么时间交付什么（Deliverable）**。
- 避免“六拍、四没、三边、只谈”等反模式（见 `references/checklists.md`）。
