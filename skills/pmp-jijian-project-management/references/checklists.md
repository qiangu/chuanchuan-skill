# 清单与反模式（落地用）

## A. 启动前：项目选择 3 问
- 能不能赚钱（机会/市场/财务）
- 能不能搞得定（技术/产能/约束）
- 对大家有没有好处（风险/环境/社会效益/内部激励）

## B. 反模式：六拍、四没、三边、只谈（自检用）

### 六拍（项目失控的常见剧情）
- 拍脑袋立项 → 拍肩膀任命PM → 拍胸脯承诺 → 拍桌子互相攻击 → 拍屁股走人 → 拍大腿后悔

### 四没（组织心态滑坡）
- 没问题 / 没关系 / 没办法 / 没资源

### 三边（范围不清导致混沌）
- 边设计 / 边实施 / 边修改（返工）

### 只谈（只谈单一维度）
- 初期只谈成本 / 中期只谈进度 / 后期只谈质量

**纠偏抓手（最小动作）**：
1) 立刻回到“一页任务书”把目标/范围/里程碑/预算说清  
2) 建立 WBS（可交付成果导向）并落实 Owner  
3) 建立状态报告与变更流程（书面化 + 影响分析 + 审批权限）  

## C. 三落实检查表（避免旁观者效应）
- 任务落实：每项任务只有 1 个负责人
- 人员落实：责任到人，不到部门/小组
- 组织落实：流程/平台/工具/激励一致

## D. 会议议程（最小可执行）

### 启动会（Kickoff）
- 宣布：项目正式开始（授权/节奏/基调）
- 对齐：任务书（目标/范围/里程碑/预算/成功标准）
- 明确：相关方与沟通机制、RAM/RACI、升级机制
- 约定：状态报告频率、变更流程、验收口径

### 周例会（含状态报告）
- 复盘上周：完成/未完成/原因
- 看偏差：范围/进度/成本/质量（是否需要纠偏/变更）
- 风险/问题：Top 5（Owner + 截止日）
- 本周计划：工作包级别承诺
- 需要决策：明确谁拍板、何时给答复

### 变更控制会（CCB）
- 变更事实确认（先定义问题）
- 影响分析（十掌全覆盖）
- 方案对比（上/中/下策）
- 审批结论（权限正确、记录完整）
- 更新基准与沟通对象

### 收尾会
- 验收状态（签收/遗留/补救计划）
- 收尾报告要点（偏差、客户关系、风险总结）
- 经验教训（写成可执行检查项）
- 归档与复用机制（谁负责、怎么查、怎么用）
